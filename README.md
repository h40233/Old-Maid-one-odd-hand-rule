# 抽鬼牌流程順暢性與勝率偏差研究

## 摘要

抽鬼牌（Old Maid）的傳統規則以一張無法配對的落單牌作為敗負條件，玩家依序抽牌並丟棄成對手牌，最後持有落單牌者為敗者。本文的研究動機來自實際遊玩經驗：當玩家在非自己行動階段被抽走最後一張牌而被動出局時，雖然可依規則跳過該玩家並繼續遊戲，但輪序判斷會短暫中斷，造成流程不連續與遊玩體驗下降。本文研究一種以奇偶性為核心的流程調整模型：令場上恰有一位奇數手牌玩家，且每回合由該玩家抽取下一位仍有手牌玩家的一張牌。本文先證明在 `T = 2kN + 1` 的總牌數模型下，此規則可維持「唯一奇數手牌玩家」的不變量，並保證玩家只會在自己的行動後出局，不會因被抽牌而被動出局。接著以 `N = 2` 至 `13` 位玩家、每組 `10,000,000` 局蒙地卡羅模擬，檢驗唯一奇數手牌玩家是否承受敗率劣勢，並比較 `T = 2kN + 1` 與 `T = 2kN - 1` 兩種發牌模型。結果顯示，`T = 2kN + 1` 中唯一奇數手牌玩家具有穩定且顯著的敗率劣勢；改用 `T = 2kN - 1`，讓唯一奇數手牌玩家少一張牌，可在多數玩家數下大幅降低敗率偏差，但偏差仍未完全消除。因此，`T = 2kN - 1` 可視為提升相對勝率公平的改良方案，而非嚴格意義下的完全公平解。

關鍵詞：抽鬼牌、Old Maid、奇偶不變量、蒙地卡羅模擬、勝率公平、程序公平

## 1. 研究背景

本研究的出發點並非單純追求數學上的勝率公平，而是實際遊玩時遇到的流程痛點。與朋友遊玩抽鬼牌時，若某位玩家起始手牌為奇數，或在初始整理後處於奇數手牌狀態，該玩家可能在被上家抽牌時失去最後一張牌。此時該玩家尚未輪到自己抽牌，卻已經在非自己行動階段出局，桌面上自然會出現疑問：下一回合究竟應由誰抽牌？原本應由該玩家抽下一家的流程是否要跳過？

後續查閱規則可知，抽鬼牌的回合推進通常不是以「自己有沒有被抽過牌」為判準，而是以「輪到自己時，若仍有手牌，就必須抽下一位玩家的牌」為判準。因此，即使有人被動出局，遊戲仍可透過跳過已無手牌玩家而繼續。然而，這種處理只解決了遊戲能否繼續的問題，並未消除體驗上的中斷感。每當有人在非自己行動階段突然出局，玩家仍需停下來確認輪序，遊戲節奏被打斷，也降低遊玩體驗。

抽鬼牌是一種以配對與抽牌為核心的家庭紙牌遊戲。常見規則使用標準 52 張牌移除一張皇后，形成 51 張牌，或使用標準 52 張牌加入一張鬼牌（joker），形成 53 張牌。玩家發完牌後先丟棄手中的成對牌，之後依序從其他玩家手中抽牌並丟棄新形成的對子，最後持有無法配對的落單牌者為敗者。Pagat 與 Bicycle Cards 對 Old Maid 的規則說明皆指出，遊戲目標是丟棄成對手牌並避免最後持有落單牌；發牌時玩家手牌數不一定相同，所有玩家完成配對後，最後留下落單牌者輸掉遊戲。

因此，傳統規則的問題不在於能否完成遊戲，而在於流程是否順暢。若一位玩家在被其他玩家抽牌時剛好失去最後一張牌，該玩家是在非自己行動階段出局，輪序必須跳接到下一位仍有牌的玩家。這種被動出局並不違反傳統規則，但會造成遊玩體驗上的中斷感，也使流程分析不夠乾淨。本文將此問題稱為程序公平問題，並將「各玩家最終敗率是否接近」稱為勝率公平問題。兩者相關但並不等同：一套規則可能流程順暢，卻仍使特定玩家具有較高敗率。

本文所研究的核心問題如下。

1. 對於 `N` 位玩家，是否可設計總牌數，使開局後恰有一位奇數手牌玩家，其他玩家皆為偶數手牌？
2. 在唯一奇數手牌玩家行動規則下，是否可保證遊戲過程中不會發生被動出局？
3. 若流程順暢性成立，唯一奇數手牌玩家是否在勝率上承受劣勢？
4. 若唯一奇數手牌玩家有敗率偏差，改用 `T = 2kN - 1` 是否能改善此偏差？

## 2. 模型與定義

令 `N` 為玩家數，`P` 為成對牌的對數，另有一張落單牌。總牌數為：

```text
T = 2P + 1
```

本文主要討論兩種模型。

第一種為奇數手牌多一張模型：

```text
P = kN
T = 2kN + 1
發牌型態：2k + 1, 2k, 2k, ..., 2k
```

第二種為奇數手牌少一張模型：

```text
P = kN
T = 2kN - 1
發牌型態：2k - 1, 2k, 2k, ..., 2k
```

兩種模型皆固定座位 0 為開局唯一奇數手牌玩家。玩家收到手牌後先丟棄所有可配對的牌。由於每次丟棄皆移除兩張同點數牌，手牌數的奇偶性不會因初始整理而改變。

本文採用以下行動規則。

1. 每回合由場上唯一奇數手牌玩家行動。
2. 行動玩家從下一位仍有手牌的玩家手中隨機抽取一張牌。
3. 若抽到的牌可與行動玩家手中某張牌配對，立即丟棄該對牌。
4. 手牌歸零者出局，不再參與後續抽牌。
5. 最後持有落單牌者為敗者。

## 3. 流程順暢性的奇偶證明

### 3.1 唯一奇數手牌玩家的存在

在 `T = 2kN + 1` 模型中，發牌型態為一位玩家 `2k + 1` 張，其餘 `N - 1` 位玩家各 `2k` 張。因此，開局恰有一位玩家手牌數為奇數。

在 `T = 2kN - 1` 模型中，發牌型態為一位玩家 `2k - 1` 張，其餘 `N - 1` 位玩家各 `2k` 張。因此，開局同樣恰有一位玩家手牌數為奇數。

初始整理階段只會移除成對的兩張牌，因此不改變任何玩家手牌數的奇偶性。故整理後仍恰有一位奇數手牌玩家。

### 3.2 奇偶傳遞不變量

假設某回合開始時，玩家 `A` 是唯一奇數手牌玩家，玩家 `B` 是下一位仍有手牌的玩家。由於場上只有 `A` 為奇數手牌玩家，因此 `B` 若仍有手牌，必為偶數手牌玩家。

當 `A` 從 `B` 手中抽一張牌後，`B` 的手牌數由偶數減一，變為奇數：

```text
B：偶數 - 1 = 奇數
```

玩家 `A` 則先由奇數加一，變為偶數：

```text
A：奇數 + 1 = 偶數
```

若抽到的牌與 `A` 原有手牌形成一對，`A` 會再丟棄兩張牌。移除兩張牌不改變偶數性，因此 `A` 仍為偶數手牌玩家。故每回合結束後，奇數手身分由 `A` 傳遞給 `B`，場上仍恰有一位奇數手牌玩家。

### 3.3 避免被動出局

在回合開始時，被抽牌者 `B` 必為偶數手牌且仍有手牌，因此其手牌數最小可能值為 2。被抽走一張牌後，最小剩餘張數為：

```text
2 - 1 = 1
```

因此，被抽牌者不可能因被抽牌而手牌歸零。玩家只可能在自己行動後，因抽到牌並形成一對、丟棄兩張牌而手牌歸零。此性質即本文所稱的程序順暢性或主動出局性。

## 4. 模擬設計

本文使用專案中的兩支 Python 模擬腳本：

```text
simulate_old_maid.py
simulate_old_maid_odd_minus.py
```

前者模擬 `T = 2kN + 1`，後者模擬 `T = 2kN - 1`。兩者皆採用相同的抽牌規則、配對規則、出局規則與統計方法。

模擬參數如下。

```text
玩家數 N：2 至 13
最大可用對數：26
k：floor(26 / N)
每組模擬局數：10,000,000
隨機種子：20260620
```

主要觀察指標包括：

1. 奇數手牌玩家敗率。
2. 其他玩家平均敗率。
3. 奇數手牌玩家敗率與其他玩家平均敗率之差。
4. 首位出局率。
5. 平均名次。
6. 被動出局次數。
7. 奇偶不變量違反次數。
8. 結局落單牌檢查錯誤次數。

本文將敗率偏差定義為：

```text
Δ = 奇數手牌玩家敗率 - 其他玩家平均敗率
```

若 `Δ > 0`，代表唯一奇數手牌玩家比其他玩家平均更容易成為最後持有落單牌者。由於每局只有一位敗者，若所有座位完全公平，則任一座位理論敗率為 `1 / N`。

## 5. 模擬結果

### 5.1 流程檢查

在 `T = 2kN + 1` 與 `T = 2kN - 1` 兩種模型中，所有 `N = 2` 至 `13` 的設定皆得到下列結果：

```text
被動出局次數：0
奇偶不變量違反次數：0
結局落單牌檢查錯誤次數：0
```

此結果與第三節的奇偶證明相符，顯示兩種模型都能達成本文定義的流程順暢性。

表 1 顯示，在 `T = 2kN + 1` 中，所有玩家數下 `Δ+` 皆為正，代表唯一奇數手牌玩家的敗率高於其他玩家平均敗率。此結果支持以下判斷：若讓唯一奇數手牌玩家起手多一張牌，該玩家會承受穩定的敗率劣勢。

改用 `T = 2kN - 1` 後，所有玩家數下 `Δ-` 仍為正，但數值明顯小於 `Δ+`。除了 `N = 3` 的改善幅度僅約 2.2%，以及 `N = 4` 約 21.4% 外，其餘多數玩家數的偏差降低幅度約為六成至九成。尤其在 `N = 9` 至 `13` 的設定中，偏差降低比例皆超過八成。

## 6. 討論

### 6.1 程序公平與勝率公平的分離

本文最重要的觀察是，程序公平與勝率公平必須分開討論。奇偶不變量能保證唯一奇數手牌玩家規則下不會發生被動出局，這是一個可由數學直接證明的流程性質。模擬結果也顯示，在兩種模型下皆未出現被動出局、奇偶不變量違反或結局檢查錯誤。

然而，流程順暢不代表勝率完全公平。`T = 2kN + 1` 雖可保證程序順暢，卻使唯一奇數手牌玩家同時承受兩個可能影響敗率的因素：起手多一張牌，以及先行動。模擬結果顯示，這兩個因素綜合後造成穩定的敗率劣勢。

### 6.2 為何 `T = 2kN - 1` 能改善偏差

在 `T = 2kN + 1` 中，唯一奇數手牌玩家起手 `2k + 1` 張，其初始持有落單牌的機率為：

```text
(2k + 1) / (2kN + 1)
```

其他任一玩家的初始持有落單牌機率為：

```text
2k / (2kN + 1)
```

因此，唯一奇數手牌玩家在起點上更可能持有落單牌。

在 `T = 2kN - 1` 中，唯一奇數手牌玩家起手 `2k - 1` 張，其初始持有落單牌的機率下降為：

```text
(2k - 1) / (2kN - 1)
```

其他任一玩家則為：

```text
2k / (2kN - 1)
```

這個調整反向補償了唯一奇數手牌玩家的先行動與奇偶角色，使敗率偏差大幅縮小。模擬結果顯示，此補償在多數玩家數下有效，但並非精準抵銷。

### 6.3 `N = 3` 的例外性

在 `N = 3` 時，`T = 2kN - 1` 對敗率偏差的改善幅度僅約 2.2%，明顯低於其他多數玩家數。這表示偏差不只由起手牌數決定，也受到玩家數、輪序、落單牌流動路徑與配對機會共同影響。後續研究可針對小玩家數進行更細緻的狀態轉移分析。

### 6.4 本研究限制

本文仍有以下限制。

1. 模擬固定座位 0 為唯一奇數手牌玩家，並以其他玩家平均敗率作為比較基準。此設計能回答奇數手牌角色是否不利，但尚未完整拆解座位方向造成的差異。
2. 本文使用 `k = floor(26 / N)` 作為標準 52 張牌限制下的取值。若改用其他 `k` 或非標準牌組，偏差大小可能不同。
3. 本文的 `T = 2kN - 1` 模型證明其可降低偏差，但尚未證明存在能使偏差完全歸零的封閉公式。
4. 本文採用隨機抽牌策略，未討論玩家具有記憶、策略或心理判斷時的結果。

## 7. 結論

本文研究一種以唯一奇數手牌玩家為核心的抽鬼牌流程調整規則。理論上，當開局恰有一位奇數手牌玩家，且每回合由該玩家抽取下一位仍有手牌玩家的一張牌時，奇數手牌身分會在玩家間傳遞，並形成穩定的不變量。由於被抽牌者在回合開始時必為偶數且仍有手牌，其手牌數最少為 2，因此不可能因被抽一張牌而被動出局。此規則可保證玩家只會在自己的行動後出局，達成程序順暢性。

勝率方面，`10,000,000` 局模擬顯示，`T = 2kN + 1` 模型下唯一奇數手牌玩家在 `N = 2` 至 `13` 的所有設定中皆具有較高敗率。此結果支持「唯一奇數手牌若起手多一張，會承受穩定敗率劣勢」的結論。

若改用 `T = 2kN - 1`，使唯一奇數手牌玩家起手少一張，敗率偏差在多數玩家數下大幅降低。尤其在 `N = 5` 以上的大多數設定中，偏差降低幅度約為六成至九成。可是，`T = 2kN - 1` 的偏差仍在所有模擬設定中維持正值，且近似 95% 信賴區間皆未包含 0。因此，本文不主張該模型完全消除勝率偏差，而主張它能顯著改善偏差，形成相對勝率公平。

綜合而言，`T = 2kN + 1` 是一個能保證程序順暢性的清楚模型；`T = 2kN - 1` 則是在保留程序順暢性的前提下，進一步改善勝率公平的實用修正。若未來研究能結合座位輪替、精確狀態轉移模型與最佳化搜尋，或可進一步尋找更接近完全勝率公平的總牌數與發牌型態。
